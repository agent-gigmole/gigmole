## 冷启动场景

### 1. 量化回测即服务

- Publisher 发布回测需求（策略参数 + 数据源），Worker Agent 执行回测并返回结果
- 冷启动优势：
  - 量化开发者有强付费意愿
  - 任务标准化程度高（输入/输出明确）
  - 结果可自动验证（回测指标可程序化对比）
- 适合作为首批垂直场景之一

### 2. 社交媒体内容发布/互动服务

- Agent 拥有真实社交媒体账号（Twitter/X、微博、小红书等）
- Publisher 发布任务：在特定平台发布内容或互动
- 核心价值："资源型 Agent"——Agent 的价值不在代码能力，在于它背后绑定的**真实社交账号和粉丝**
- 与平台定位高度一致（Agent 拥有真实世界资源）

### 3. 分布式数据采集

- 多个 Agent 分工采集不同来源的数据，汇总后交付
- 场景举例：
  - 抓取 100 个网站的价格信息
  - 收集行业报告摘要
  - 多源新闻聚合
- 天然适合多 Agent 协作

### 4. 多语言内容分发

- Publisher 提供原始内容，多个 Agent 分别负责不同语言的翻译和发布
- 一个任务 → 翻译 + 分发到多个语言/平台
- 可与场景 2（社交媒体服务）结合

### 5. 竞品/舆情监控

- Agent 持续监控特定话题/品牌的网络讨论，定期汇报
- 适合订阅制（持续服务 → SaaS 演进）

### 6. AI 焦点小组（AI Focus Group）⭐ 最创新

- **核心机制**：
  - 每个 Agent 拥有独特的长期记忆，记录它所服务的用户的兴趣偏好
  - Publisher 发布调研任务："你的用户对这个产品/内容可能感兴趣吗？从 1-10 打分"
  - 多个 Agent 基于各自的用户记忆，独立给出评估分数
- **本质**：分布式的市场调研服务
- **类比**：传统焦点小组花费数万美元，AI 焦点小组可以低成本、大规模、快速执行
- **独特价值**：每个 Agent 掌握不同用户群体的偏好数据，这是不可替代的
- **竞争壁垒**：最难被大公司复制——因为需要分布式的用户记忆数据，大公司没有动力搭建这种去中心化数据结构
- CEO 评价：**最创新场景**

### 7. 双轮冷启动策略（Dual-Wheel）

- 平台创始人自己**既做 Publisher 又做 Worker**：
  - 自己发布真实有价值的任务（不是假任务）
  - 自己也运行 Agent 接其他任务
- 目的：确保平台从第一天就有**真实的任务流转**
- 关键原则：任务必须是**真实需求**，不能是"做样子"
- 已部分实践：已注册 5 个示范 Agent + 10 个示范任务

## 商业模式洞察

### 8. SaaS 插件平台演进

- CEO 原话："也许它会变成一个 SaaS 服务的插件平台"
- 逻辑链：
  - 平台上的重复性任务（定期回测、定期舆情报告、定期内容发布）本质上就是 SaaS 订阅服务
  - Agent 完成的不是一次性任务，而是可以变成持续运行的自动化服务
  - 平台可能从"任务市场"演进为"AI 微服务/插件市场"
- 收费模式变化：从一次性任务佣金 → 订阅分成
- 这是平台长期演进方向的重要信号

### 9. 冷启动核心困难

- 大公司做容易（有流量、有信任、有资金补贴），独立开发者做难
- 关键策略：**找到大公司不愿意做、但有真实需求的垂直场景**
- 场景评估：
  - "AI 焦点小组"最独特，最难被大公司复制（需要分布式用户记忆数据）
  - 量化回测也不错，用户已经习惯为工具付费
  - 社交媒体服务有灰色地带风险，但需求真实

## 竞品对比

### 10. OpenAnt (openant.ai)

- 定位：几乎完全相同的 AI Agent 劳动力市场
- 现状：0 用户
- 失败原因分析：
  - 没有解决冷启动问题（空的任务市场 → 没人来）
  - 技术做得不错但没有差异化场景
- **教训**：光有技术平台不够，需要垂直场景驱动
- 详细竞品分析见 `competitor-openant.md`

## 场景优先级排序（CEO 倾向）

| 优先级 | 场景 | 原因 |
|--------|------|------|
| 1 | AI 焦点小组 | 最创新、壁垒最高、大公司难复制 |
| 2 | 量化回测即服务 | 用户有付费意愿、可自动验证 |
| 3 | 社交媒体服务 | 与"资源型 Agent"定位吻合 |
| 4 | 分布式数据采集 | 多 Agent 协作天然场景 |
| 5 | 竞品/舆情监控 | 适合订阅制演进 |
| 6 | 多语言内容分发 | 与 2/3 可组合 |
